# 发布旅程{#concept_mtc_lrt_52b}

您可以在测试旅程的有效性后发布该旅程。

如果您需要对已发布的旅程进行修改，则需要创建旅程的新版本。 请参见 [](../building-journeys/journey-versions.md)。当旅程为只读时，您只能修改活动标签和说明、旅程名称和旅程说明。

如果您停止某个旅程，它将永久停止。 所有的人都将被永久地拦住，而且旅程将停止允许新的入口。 如果您需要再次使用旅程，您需要复制并发布它。

1. 在发布您的旅程之前，请确认该旅程有效且没有错误。 您将无法发布包含错误的旅程。 请参见 [](../about/troubleshooting.md#section_h3q_kqk_fhb)。还建议在发布前测试您的旅程。 请参见 [](../building-journeys/testing-the-journey.md)。
1. 要发布旅程，请单击位 **[!UICONTROL Publish]**于右上方下拉菜单中的选项。

   ![](../assets/journeyuc1_18.png)

发布旅程后，它将处于只读模式。

以下是个人旅程的不同状态：

* _运行_:这个人目前正在旅程中。
* _完成_:这个人正在旅程的尽头。
* _错误_:由于出错，个人在旅途中被停止。
* _超时_:由于一步太费时间，这个人在旅途中被拦住了。
